# Changelog - Prince of Persia Classic (PS Vita Port)

Historial de versiones de la build (`VITA_VERSION` en `CMakeLists.txt`). Cada entrada apunta a la sección
correspondiente de `Fixes_Log.md` y `plan_portabilidad.md` para el detalle técnico completo. Todas las
builds desde v01.10 comparten el mismo `TITLEID` (`POPC00001`), así que instalar una nueva versión sobre
otra conserva las partidas guardadas.

---

### 🇪🇸 Español

| Versión | VPK | Resumen |
|---|---|---|
| 01.00 | `popclassic.vpk` | Baseline original: port jugable de punta a punta (touch, caminar, saltar, agacharse confirmados en hardware real), **sin audio** (`audio.cpp` con stubs "Audio disabled for stability"). |
| 01.10 | `popclassic_soloud.vpk` | Primer intento de audio real con **SoLoud**: BGM streaming, SFX cacheados, sanitizador de rutas, dummy handles. Crasheó 11 veces en consola al abrir el juego (antes del menú). |
| 01.11 | `popclassic_audio.vpk` | **SoLoud eliminado.** Causa raíz de los crashes: SoLoud esperaba un `stb_vorbis` compilado con su "file hack" y el proyecto lo compiló crudo contra newlib real — cada carga de BGM ejecutaba fseek/ftell sobre un objeto C++ disfrazado de FILE. Reemplazado por un **mixer propio sobre `sceAudioOut`** (patrón deadspace-vita): BGM streaming con `stb_vorbis_open_memory`, SFX con `stb_vorbis_decode_memory`, 12 voces con resampling lineal, todo IO por `sceIo` (sin stdio). También se agrandó `__sF_fake` (bug independiente que rociaba texto sobre `.data`). Ver Fixes_Log §8–10. |
| 01.12 | `popclassic_audio.vpk` | Tres mejoras sobre la base estable: **(a)** logging verboso siempre activo — causa raíz del "solo veo crashes": los `l_debug`/`l_info` solo se compilaban con `CMAKE_BUILD_TYPE=Debug`, pero el script de build siempre usa `Release`. **(b)** audio menos distorsionado/bajo: se quitó `sceAudioOutSetVolume` redundante, se dobló el buffer de mezcla (1024→2048 frames) y se cambió el hard-clip por un limitador suave (`tanh`). **(c)** reproducción de video real con **SceAvPlayer** nativo (antes solo se saltaba el video). Ver Fixes_Log §11. |
| 01.13 | `popclassic_audio.vpk` | **Regresión de v01.12:** activar `FALSOJNI_DEBUGLEVEL=0` para el logging terminó logueando *cada primitiva JNI interna* (NewStringUTF, FindClass, GetEnv, DeleteLocalRef...) cientos de veces por segundo, cada una con un ciclo completo de abrir/escribir/cerrar archivo — el juego se volvió injugable y el audio sonó peor (el hilo de mezcla se quedaba sin CPU). Fix: no tocar `FALSOJNI_DEBUGLEVEL` (FalsoJNI no tiene logs útiles en su nivel por defecto) + mantener el archivo de log abierto entre líneas en vez de abrir/cerrar por línea. Ver Fixes_Log §12. |
| 01.14 | `popclassic_audio.vpk` | **v01.13 seguía lento.** Causa: una condición pre-existente del motor (no introducida en este trabajo) dispara errores WARN/ERROR de FalsoJNI (`GetArrayLength`/`GetIntArrayRegion`/`GetFloatArrayRegion`, siempre con `length=0`, un no-op válido en JNI real) en *cada frame con toque activo* — antes invisibles (solo consola), ahora escritos a disco en cada frame al rutear FalsoJNI al logger de archivo. Arreglado en la fuente (esos casos ya no loguean). De paso se restauró `preloadEffect`, que se había quitado por error de una tabla JNI durante la investigación del crash de SoLoud. Se agregó también supresión de líneas duplicadas consecutivas en el logger como defensa adicional. Ver Fixes_Log §13. |
| 01.15 | `popclassic_audio.vpk` | El log de "input tick" en `main.c` disparaba en *cada frame* con cualquier toque activo (por nivel), no solo cuando el estado cambiaba — la deduplicación de v01.14 solo colapsa mensajes idénticos consecutivos, así que el jitter de `touch.reportNum` entre frames lo esquivaba. Fix: el log ahora es *edge-triggered* (solo escribe cuando el estado realmente cambia, más un latido cada 120 frames en reposo). Ver Fixes_Log §14. |
| 01.16 | `popclassic_audio.vpk` | Fix de volumen cero en todos los SFX debido a basurilla en el stack de JNI. |
| 01.17 | `popclassic.vpk` | **Controles Físicos Puros y Fix de SceAvPlayer.** Controles: se quitó la simulación de toques virtuales para el D-Pad (que causaba clicks fantasma en los menús) y se mapeó el hardware físico directamente; joystick calibrado. Video: el reproductor de hardware de la Vita fallaba porque se le daba memoria caché (`memalign`). Reescribimos `av_alloc` para que use `sceKernelAllocMemBlock` (CDRAM/Uncached), permitiendo a la GPU leer el MP4 sin colgarse, y se quitaron wrappers innecesarios de lectura de archivos. Ver Fixes_Log §16. |
| 01.18 | `popclassic.vpk` | **Video fluido y correcto (LUTs, fix de tinte verde, fix de fuga de estado GL) y audio de cutscenes reparado.** Video: LUTs precalculadas para la conversión YUV→RGBA (sin costo de CPU, fluido); corregido el tinte verde (el decodificador entrega NV12, no I420 plano como se asumía); corregida la pantalla invertida/espejo y los cuadros negros en texturas que aparecían en el gameplay justo después de una cinemática (el dibujo del video pisaba la proyección de cocos2d y dejaba el blending apagado sin restaurarlo nunca). Audio de cutscenes: **reparado, no es un issue conocido** — el puerto se abría como tipo `MAIN` (exige 48000Hz por spec) y casi todos los videos son 44100Hz, rechazados con `SCE_AUDIO_OUT_ERROR_INVALID_SAMPLE_FREQ` (antes malinterpretado como contención de puerto); además el tamaño de paquete de audio estaba hardcodeado en 1024 en vez de leerse del frame real, causando el corte/entrecortado. Controles: Círculo ahora comparte el mismo keycode que Abajo/joystick-abajo para agacharse (antes mandaba un keycode al que el motor no reaccionaba). Se intentó (dos veces) agregar una marca visual de "presionado" al botón de Salto en el mando virtual y se revirtió ambas veces: rompía la navegación de menús por completo — pendiente de una señal real de estado menú/gameplay que hoy no existe en el código. Ver Fixes_Log §17. |

### 🇬🇧 English

| Version | VPK | Summary |
|---|---|---|
| 01.00 | `popclassic.vpk` | Original baseline: fully playable end-to-end port (touch, walk, jump, crouch confirmed on real hardware), **no audio** (`audio.cpp` stubbed with "Audio disabled for stability"). |
| 01.10 | `popclassic_soloud.vpk` | First real-audio attempt using **SoLoud**: streamed BGM, cached SFX, path sanitizer, dummy handles. Crashed 11 times on console right at launch (before the menu). |
| 01.11 | `popclassic_audio.vpk` | **SoLoud removed.** Root cause of the crashes: SoLoud expected an `stb_vorbis` built with its "file hack", but the project compiled it raw against real newlib -- every BGM load ran fseek/ftell over a C++ object disguised as a FILE. Replaced with a **custom mixer over `sceAudioOut`** (deadspace-vita pattern): BGM streamed via `stb_vorbis_open_memory`, SFX via `stb_vorbis_decode_memory`, 12 voices with linear resampling, all IO through `sceIo` (no stdio). Also grew `__sF_fake` (an independent bug that sprayed text over `.data`). See Fixes_Log #8-10. |
| 01.12 | `popclassic_audio.vpk` | Three improvements on the stable base: **(a)** verbose logging always on -- root cause of "I only see crashes": `l_debug`/`l_info` only compiled in with `CMAKE_BUILD_TYPE=Debug`, but the build script always uses `Release`. **(b)** less distorted/quiet audio: removed a redundant `sceAudioOutSetVolume` call, doubled the mix buffer (1024->2048 frames), replaced the hard clip with a soft (`tanh`) limiter. **(c)** real cutscene playback via native **SceAvPlayer** (previously videos were just skipped). See Fixes_Log #11. |
| 01.13 | `popclassic_audio.vpk` | **v01.12 regression:** enabling `FALSOJNI_DEBUGLEVEL=0` for logging ended up logging *every primitive JNI call* (NewStringUTF, FindClass, GetEnv, DeleteLocalRef...) hundreds of times per second, each paying a full file open/write/close cycle -- the game became unplayable and audio got worse (the mixer thread was starved of CPU). Fix: don't touch `FALSOJNI_DEBUGLEVEL` (FalsoJNI has no useful logs at its default level) + keep the log file open across lines instead of opening/closing per line. See Fixes_Log #12. |
| 01.14 | `popclassic_audio.vpk` | **v01.13 was still slow.** Cause: a pre-existing engine condition (not introduced by this work) triggers FalsoJNI WARN/ERROR calls (`GetArrayLength`/`GetIntArrayRegion`/`GetFloatArrayRegion`, always with `length=0`, a valid no-op under real JNI) on *every touch-active frame* -- previously invisible (console-only), now written to disk every frame once FalsoJNI was routed to the file logger. Fixed at the source (those cases no longer log). Also restored `preloadEffect`, which had been mistakenly removed from a JNI table during the SoLoud crash investigation. Added consecutive-duplicate suppression to the logger as extra insurance. See Fixes_Log #13. |
| 01.15 | `popclassic_audio.vpk` | The "input tick" log in `main.c` fired on *every frame* with any active touch (level-triggered), not only when the state changed -- v01.14's deduplication only collapses exactly-identical consecutive messages, so jitter in `touch.reportNum` between frames dodged it. Fix: the log is now *edge-triggered* (only writes when the state actually changes, plus a heartbeat every 120 frames while idle). See Fixes_Log #14. |
| 01.16 | `popclassic_audio.vpk` | Fixed zero-volume bug on all SFX (footsteps, hits, environment) caused by uninitialized JNI stack memory. |
| 01.17 | `popclassic.vpk` | **Pure Physical Controls & SceAvPlayer Fix.** Controls: disabled synthetic touch events for the D-Pad (which caused ghost clicks in menus) and mapped physical buttons properly; calibrated analog stick. Video: Vita's hardware media engine aborted because it was fed cached memory via `memalign`. Rewrote `av_alloc` to use native `sceKernelAllocMemBlock` (CDRAM/Uncached), allowing the hardware to read the MP4 frames without crashing, and removed problematic file I/O wrappers. See Fixes_Log #16. |
| 01.18 | `popclassic.vpk` | **Video now fluid AND correct (LUTs, green-tint fix, GL state leak fix) plus a real cutscene audio fix.** Video: precomputed LUTs for YUV->RGBA (no CPU cost, fluid); fixed the green tint (the decoder outputs NV12, not planar I420 as assumed); fixed the mirrored/upside-down screen and black-box sprites that appeared in gameplay right after a cutscene (the video draw call was overwriting cocos2d's projection and left blending disabled without ever restoring it). Cutscene audio: **actually fixed, not a known issue anymore** -- the port was opened as type `MAIN` (which the spec requires to be exactly 48000Hz) while almost every video is 44100Hz, rejected with `SCE_AUDIO_OUT_ERROR_INVALID_SAMPLE_FREQ` (previously misread as port contention); the audio chunk length was also hardcoded to 1024 instead of read from the real frame, causing the choppiness. Controls: Circle now shares the same keycode as Down/left-stick-down for crouch (it previously sent a keycode the engine never reacted to). A "pressed" visual highlight for the on-screen Jump button was attempted (twice) and reverted both times: it broke menu navigation entirely -- pending a real gameplay/menu state signal that doesn't exist in the code yet. See Fixes_Log #17. |
